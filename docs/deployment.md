# 🚀 الدليل التشغيلي (Deployment & Docker)

المشروع مبني ليكون معزولاً بالكامل داخل حاويات Docker لضمان استقراره، ولكنه يدعم التشغيل المباشر (Bare-metal) والتشغيل المحلي بـ HTTPS.

---

## 1. بيئة التطوير باستخدام Docker (Development)
ملف `docker-compose.dev.yml` مصمم لتوفير تجربة تطوير سريعة:
- **Hot-Reloading:** أي تعديل يتم تطبيقه فوراً.
- **مزامنة البيانات:** عند التشغيل، يعمل الأمر `npx prisma db push` لمزامنة قاعدة البيانات.
- **المنفذ (Port):** التطبيق يعمل على المنفذ `3000` وقاعدة البيانات على `5432`.

**أمر التشغيل:**
```bash
docker compose -f docker-compose.dev.yml up --build
```

---

## 2. التشغيل المحلي ببروتوكول HTTPS (Local HTTPS)
للتأكد من عمل الميزات المتقدمة (مثل Service Workers وتطبيقات PWA) في بيئة التطوير المحلية، يجب تشغيل المشروع بـ `https://localhost`.

### الطريقة الأسهل (Next.js المدمجة):
توفر Next.js ميزة توليد شهادات HTTPS محلياً بشكل تلقائي باستخدام أداة `mkcert` (تعمل خلف الكواليس):
```bash
npm run dev -- --experimental-https
```
*(ملاحظة: عند تشغيل الأمر لأول مرة، سيطلب منك النظام صلاحيات الإدارة/كلمة المرور لتثبيت شهادة الثقة في جهازك).*

### طريقة Docker (Caddy / Nginx):
إذا كنت تستخدم بيئة Docker بالكامل وتود تفعيل HTTPS محلياً، يمكنك إضافة خدمة `Caddy` كـ Reverse Proxy في `docker-compose.dev.yml`، حيث يقوم Caddy بتوليد شهادات SSL محلية مجانية وتوجيه الحركة إلى الحاوية الأساسية.

---

## 3. بيئة الإنتاج باستخدام Docker (Production)
- **Next.js Standalone:** يتم استخراج الملفات الأساسية فقط (أقل من `150MB`).
- **الترحيل الآمن:** الحاوية تنفذ `npx prisma migrate deploy` تلقائياً لتحديث قاعدة البيانات.
- **الأمان:** منفذ قاعدة البيانات مغلق (غير مكشوف للإنترنت).

**أمر التشغيل للإنتاج:**
```bash
docker compose up -d --build
```

---

## 4. التخزين الآمن (Volumes)
لتفادي مسح البيانات عند إعادة تشغيل الـ Containers، تم تخصيص الـ Volumes التالية:
1. **`postgres_data`**: لحفظ الجداول والطلبات.
2. **`uploads_data`**: مربوط بمسار `/app/public/uploads` للحفاظ على الصور والملفات المرفوعة آمنة.

---

## 5. الدليل الشامل للرفع المباشر على سيرفر (VPS Ubuntu)

إذا كنت تفضل تشغيل المشروع بشكل مباشر (Bare-Metal) دون Docker لضمان أقصى أداء ممكن من السيرفر، فقد قمنا بتوفير دليل مفصل خطوة بخطوة يشمل إعداد (Nginx, PM2, PostgreSQL) وتأمين الاتصال (SSL).
