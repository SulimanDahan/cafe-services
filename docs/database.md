# 💾 قواعد البيانات (Database & Prisma)

يعتمد النظام على **PostgreSQL** لإدارة البيانات، مع استخدام **Prisma ORM** لتسهيل الاستعلامات وضمان سلامة الأنواع (Type-Safety).

## 1. تصميم المخطط (Schema)

ملف `prisma/schema.prisma` يحتوي على الهيكل الأساسي للنظام، ويشمل:

- **المستخدمين (Users):** إدارة تسجيل الدخول والصلاحيات (مدير / مستخدم عادي).
- **المنتجات (Items & Item Groups):** تصنيفات وقوائم الطعام والمشروبات.
- **الطلبات (Orders):** إدارة الفواتير والمبيعات اليومية.
- **الحجوزات (Reservations):** إدارة حجوزات الغرف مع حالات متعددة (معلق، نشط، مرفوض).

## 2. إدارة الاتصال (Singleton Client)

> [!CAUTION]
> **منع تسرب الذاكرة (Memory Leak):**
> في بيئة `Next.js`، استدعاء `new PrismaClient()` في كل صفحة يؤدي إلى استنفاد الـ Connection Pool وتوقف قاعدة البيانات.
> لذلك، **يجب دائماً** استدعاء `prisma` من ملف `@/lib/prisma`.

```typescript
// ✅ الطريقة الصحيحة للاستيراد
import { prisma } from "@/lib/prisma";
```

## 3. عمليات التطوير (Workflows)

### أثناء التطوير (Prototyping)

عند إجراء أي تغيير على `schema.prisma`، يتم استخدام `db push` لمزامنة القاعدة محلياً دون توليد ملفات تهجير معقدة. يتم ذلك تلقائياً عند تشغيل حاوية الـ Development.

### في بيئة الإنتاج (Production)

لا يتم استخدام `db push` إطلاقاً، بل يتم الاعتماد على `npx prisma migrate deploy` لتشغيل ملفات التهجير وتحديث قاعدة البيانات دون إتلاف أي بيانات سابقة.

## 4. النسخ الاحتياطي والاستعادة (Backup & Clean Restore)

عند نقل المشروع إلى سيرفر جديد، قد تقوم الحاوية بإنشاء مستخدم وإعدادات افتراضية. لتفادي حدوث أي تعارض ولضمان عملية استعادة نظيفة، تم توفير أوامر لمسح قاعدة البيانات الافتراضية وحقن النسخة الاحتياطية.

### أخذ نسخة احتياطية من السيرفر الحالي:

لتوليد ملف نسخة احتياطية باسم `backup.sql` من حاوية الإنتاج:

```bash
docker exec -t cafe-postgres-prod pg_dump -U cafe_admin -d cafe_db > backup.sql
```

### استعادة النسخة الاحتياطية على السيرفر الجديد:

1. ضع ملف `backup.sql` في المجلد الرئيسي للمشروع على السيرفر الجديد.
2. قم بتشغيل الأمر التالي (يقوم بمسح الـ Schema الافتراضية بالكامل وإعادة إنشائها نظيفة ثم استيراد بياناتك الاحتياطية):

- **في بيئة الإنتاج (Production):**

    ```bash
    npm run db:restore
    ```

- **في بيئة التطوير (Development):**
    ```bash
    npm run db:restore:dev
    ```
