# الدليل الشامل لاستضافة التطبيق محلياً على نظام Windows (بدون Docker)

هذا الدليل يشرح بالتفصيل الممل رحلة استضافة النظام من الصفر المطلق وحتى يعمل كنظام إنتاج (Production Server) حقيقي يعمل بصمت وتلقائياً **بدون أي تدخل من المستخدم** (حتى قبل تسجيل الدخول في شاشة الويندوز)، مع ضمان استقرار الإشعارات المباشرة (SSE) في شبكة الـ MikroTik.

---

## المرحلة الأولى: تجهيز البنية التحتية للسيرفر

يجب تخصيص جهاز ويندوز ويفضل تثبيت IP ثابت له من راوتر المايكروتيك (مثلاً `192.168.1.100`) لكي تتصل به أجهزة الكاشير دائماً.

### 1. تثبيت بيئة تشغيل النظام (Node.js)

- اذهب إلى موقع `nodejs.org` وحمّل إصدار **LTS** (الإصدار المستقر طويل الأمد).
- قم بتثبيته مثل أي برنامج عادي (Next -> Next).
- للتأكد من التثبيت، افتح موجه الأوامر (CMD) واكتب `node -v`، سيظهر لك رقم الإصدار.

### 2. تثبيت قاعدة البيانات (PostgreSQL)

- اذهب لـ `postgresql.org` وحمل نسخة الويندوز وثبتها.
- أثناء التثبيت، سيطلب منك تعيين **كلمة مرور (Password)** للمستخدم الرئيسي `postgres`. **احفظ هذه الكلمة جيداً** (مثلاً `123456`).
- بعد التثبيت، افتح برنامج **pgAdmin 4** (يأتي مع قاعدة البيانات).
- أدخل كلمة المرور، ثم في القائمة اليسرى، اضغط بزر الماوس الأيمن على `Databases` واختر `Create -> Database`.
- سمّ القاعدة `cafe_db` واضغط حفظ. المخزن الآن جاهز.

---

## المرحلة الثانية: إعداد المشروع وربطه بقاعدة البيانات

1. ضع مجلد المشروع `cafe-services` في مسار آمن لا يتم مسحه (مثلاً `C:\cafe-services`).
2. داخل مجلد المشروع، انسخ ملف `.env.example` وغيّر اسم النسخة إلى `.env`.
3. افتح ملف `.env` وقم بتعديل سطر `DATABASE_URL` ليتطابق مع بياناتك:
    ```env
    DATABASE_URL="postgresql://postgres:123456@localhost:5432/cafe_db?schema=public"
    ```
    _(استبدل `123456` بكلمة المرور الخاصة بك)._
4. افتح موجه الأوامر (CMD) بداخل مجلد المشروع واكتب بالترتيب:
    - `npm install` (لتحميل الحزم والمكتبات) او `npm ci` (لنسخة الproduction).
    - `npx prisma generate` (لتعريف التطبيق بهيكل قاعدة البيانات).
    - `npx prisma db push` (لإنشاء الجداول في قاعدة بيانات `cafe_db` تلقائياً).
    - `set NODE_OPTIONS=--max_old_space_size=4096 && npm run build` (لضغط أكواد التطبيق وتحويلها لنسخة "إنتاج" سريعة ومستقرة. استخدمنا `NODE_OPTIONS` لزيادة الذاكرة لـ 4 جيجابايت لتفادي خطأ `heap out of memory`).

## المرحلة الثالثة: إعداد بوابة Nginx (حارس الإشعارات)

Nginx ضروري لسببين: يتيح الوصول للتطبيق عبر الـ IP مباشرة (منفذ 80)، ويمنع انقطاع نظام الإشعارات (SSE) المباشرة.

1. حمل Nginx للويندوز من `nginx.org` وفك الضغط عنه في `C:\nginx`.
2. اذهب لمسار `C:\nginx\conf` وافتح ملف `nginx.conf` باستخدام المفكرة، وامسح محتواه بالكامل وضع الكود التالي:

```nginx
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile      on;

    keepalive_timeout  65;
    client_max_body_size 50M; # للسماح برفع صور المنتجات بحجم كبير

    server {
        listen       80;
        server_name  localhost;

        # 1. إعدادات الإشعارات المباشرة (SSE) - لمنع التقطيع
        location /api/notifications/stream {
            proxy_pass http://127.0.0.1:3000/api/notifications/stream;
            proxy_http_version 1.1;
            proxy_set_header Connection '';
            proxy_set_header Host $host;

            # إغلاق الـ Buffering هنا هو السر لمرور الإشعارات فوراً بدون حجزها
            proxy_cache off;
            proxy_buffering off;
            proxy_read_timeout 24h;
        }

        # 2. إعدادات باقي أجزاء التطبيق
        location / {
            proxy_pass http://127.0.0.1:3000;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }
    }
}
```

## المرحلة الرابعة: التشغيل التلقائي (بدون أي تدخل من المستخدم)

**الهدف:** أن يعمل النظام (تطبيق Next.js + خادم Nginx) في الخلفية فور تشغيل زر الكهرباء في الكمبيوتر، حتى ولو لم يقم أحد بكتابة كلمة مرور الويندوز لتسجيل الدخول (في شاشة القفل).
لتحقيق ذلك، سنحول البرنامجين إلى **خدمات ويندوز صامتة (Windows Services)** باستخدام أداة تُسمى **NSSM**.

### 1. تجهيز أداة NSSM

- حمل الأداة من الموقع الرسمي `http://nssm.cc/download` (ملف ZIP).
- فك الضغط عنها وضع المجلد في مسار ثابت مثل `C:\nssm`.
- افتح موجه الأوامر (CMD) **كمسؤول** (Run as administrator)، وانتقل لمسار الأداة:
    ```cmd
    cd C:\nssm\win64
    ```

### 2. جعل Nginx يعمل كخدمة تلقائية

- في شاشة الـ CMD، اكتب الأمر:
    ```cmd
    nssm install NginxService
    ```
- ستفتح نافذة رسومية، قم بإعدادها كالتالي:
    - **Path:** اضغط على `...` واختر `C:\nginx\nginx.exe`.
    - **Directory:** سيكون تلقائياً `C:\nginx`.
- اضغط على زر **Install service**.

### 3. جعل التطبيق (Next.js) يعمل كخدمة تلقائية

- في نفس الـ CMD، اكتب الأمر:
    ```cmd
    nssm install CafeAppService
    ```
- ستفتح نافذة رسومية جديدة، قم بإعدادها بحذر شديد:
    - **Path:** ابحث عن مسار أداة `npm`. عادةً يكون `C:\Program Files\nodejs\npm.cmd`. (تأكد من اختيار ملف الـ `npm.cmd`).
    - **Arguments:** اكتب بداخلها: `run start`
    - **Directory:** اختر المجلد الذي يوجد فيه مشروعك، مثلاً `C:\cafe-services`.
- اضغط على التبويب **Details** من الأعلى:
    - تأكد أن **Startup type** مضبوط على `Automatic`.
- اضغط على التبويب **I/O** (اختياري ولكنه مفيد):
    - يمكنك توجيه مخرجات التطبيق لملف لمعرفة الأخطاء، باختيار مسار مثل `C:\cafe-services\error.log` لخانتي Output و Error.
- أخيراً اضغط **Install service**.

### 4. تشغيل الخدمات لأول مرة

- من قائمة ابدأ، ابحث عن تطبيق **Services** (الخدمات) وافتحه.
- ابحث في القائمة عن `NginxService` وانقر بزر الماوس الأيمن -> **Start**.
- ابحث عن `CafeAppService` وانقر بزر الماوس الأيمن -> **Start**.

> **ملاحظة:** من الآن فصاعداً، بمجرد ضغط زر التشغيل (Power) في جهاز الويندوز، ستعمل هذه الخدمات تلقائياً من تلقاء نفسها كأرواح خفية في النظام بدون أي نوافذ، ولن تحتاج لتسجيل الدخول أبداً.

## المرحلة الخامسة: الجدار الناري (Firewall) والوصول من المايكروتيك

لكي تسمح للأجهزة بالدخول للسيرفر عبر منفذ Nginx:

1. افتح **Windows Defender Firewall with Advanced Security** من قائمة ابدأ.
2. من القائمة اليسرى، اختر **Inbound Rules** ثم من اليمين **New Rule**.
3. اختر **Port** واضغط Next.
4. اختر **TCP** واكتب في المربع **80** واضغط Next.
5. اختر **Allow the connection** ثم تتبع الخطوات حتى النهاية وسمّ القاعدة `Cafe Web 80`.

### النتيجة النهائية

من الآيباد أو جهاز الكاشير المتصل بشبكة المايكروتيك، فقط افتح المتصفح واكتب الـ IP الخاص بالسيرفر `http://192.168.1.100`، وسيعمل التطبيق بثبات مع كامل إشعاراته.

## المرحلة السادسة: تفعيل الاتصال الآمن (HTTPS/SSL) - اختياري

إذا كنت ترغب في تشغيل النظام عبر اتصال مشفر (HTTPS) لتأمين الشبكة باستخدام شهادة موقعة ذاتياً (Self-Signed)، يمكنك توليد الشهادة محلياً في ثوانٍ.

### 1. توليد الشهادات باستخدام سكربت مدمج

1. افتح موجه الأوامر (CMD) بداخل مجلد المشروع.
2. قم بتثبيت الحزمة المسؤولة عن توليد الشهادات:
    ```bash
    npm install selfsigned
    ```
3. قم بتشغيل السكربت المدمج (الموجود في مجلد المشروع):
    ```bash
    node generate-ssl.js
    ```
4. سيتم فوراً إنشاء ملفين في المجلد: `cert.pem` و `key.pem`.
5. قم بنقل (Cut) هذين الملفين إلى مجلد الـ `conf` الخاص بـ Nginx (مثلاً: `C:\nginx\conf`).

### 2. إعداد الخادم والجدار الناري

- قُم باستخدام ملف `nginx-windows.conf` (المرفق في المشروع) والذي يحتوي مسبقاً على إعدادات SSL والمنافذ 80 و 443 كبديل لملف إعدادات Nginx الحالي.
- لا تنسَ فتح المنفذ الآمن **443** (TCP) في جدار حماية الويندوز (Windows Firewall) بنفس طريقة فتحك للمنفذ 80 سابقاً.

الآن يمكن الدخول للتطبيق بأمان عبر: `https://192.168.1.100`

## المرحلة السابعة: ربط السيرفر بدومين حقيقي لحل تحذير "غير آمن" (الشهادات الذاتية)

إذا كنت ترغب في التخلص من تحذير المتصفح للشهادات الموقعة ذاتياً (Self-Signed) في شبكتك المحلية بدون أن يكون السيرفر مكشوفاً للإنترنت، يمكنك استخدام حيلة "الدومين الفعلي" مع راوتر المايكروتيك (MikroTik):

1. **شراء الدومين والحصول على الشهادة الرسمية:**
   قُم بشراء نطاق حقيقي (مثلاً `mycafe.com`)، واستخدم طريقة `DNS-01 Challenge` مع خدمة Let's Encrypt للحصول على شهادات رسمية مجانية (`fullchain.pem` و `privkey.pem`) عن طريق لوحة تحكم الدومين (بدون الحاجة لربط السيرفر المحلي بالإنترنت). ضع هذه الشهادات في مجلد `conf` الخاص بـ Nginx بدلاً من الشهادات الذاتية.

2. **تعديل إعدادات Nginx:**
   افتح ملف الإعدادات واستبدل القيم كالتالي:
   ```nginx
   server_name mycafe.com;
   ssl_certificate fullchain.pem;
   ssl_certificate_key privkey.pem;
   ```
   *(ثم أعد تشغيل خدمة Nginx).*

3. **توجيه الدومين محلياً عبر المايكروتيك (Static DNS):**
   ادخل إلى إعدادات راوتر المايكروتيك الخاص بالشبكة:
   - اذهب إلى `IP -> DNS -> Static`
   - أضف قاعدة جديدة:
     - **Name:** `mycafe.com`
     - **Address:** `192.168.1.100` (الـ IP المحلي للسيرفر).

**النتيجة السحرية:** عند كتابة `https://mycafe.com` في أجهزة الكاشير أو الآيباد داخل الكافيه، سيقوم المايكروتيك بتوجيههم فوراً للسيرفر المحلي، وسيقدم السيرفر الشهادة المعتمدة لمتصفحهم، فيظهر **القفل الأخضر** وتختفي رسالة "غير آمن" تماماً بشكل رسمي واحترافي.
