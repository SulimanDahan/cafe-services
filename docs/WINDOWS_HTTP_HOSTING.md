# الدليل الشامل لاستضافة التطبيق محلياً (Production) عبر بروتوكول HTTP فقط على Windows

هذا الدليل مخصص لتشغيل النظام في بيئة إنتاج (Production) حقيقية ومستقرة على خادم يعمل بنظام Windows داخل شبكة محلية (مثل شبكات MikroTik)، مع **إجبار النظام على العمل ببروتوكول HTTP فقط** ومنع المتصفحات والجوالات من التحول التلقائي إلى HTTPS.

---

## المرحلة الأولى: المتطلبات الأساسية وتجهيز قاعدة البيانات

1. **تثبيت Node.js:** قم بتحميل وتثبيت إصدار LTS من `nodejs.org`.
2. **تثبيت PostgreSQL:** قم بتحميل وتثبيت قاعدة البيانات من `postgresql.org` مع تذكر كلمة المرور (مثلاً `123456`).
3. **تجهيز قاعدة البيانات:** 
   - افتح أداة `pgAdmin 4`.
   - قم بإنشاء قاعدة بيانات جديدة باسم `cafe_db`.

---

## المرحلة الثانية: إعداد بيئة المشروع للإنتاج

1. ضع مجلد المشروع `cafe-services` في مسار ثابت وآمن (مثل `C:\cafe-services`).
2. قم بنسخ ملف `.env.example` وإعادة تسميته إلى `.env`.
3. افتح ملف `.env` وقم بتعديل رابط الاتصال بقاعدة البيانات `DATABASE_URL` وإضافة السماح لملفات تعريف الارتباط غير الآمنة:
   ```env
   DATABASE_URL="postgresql://postgres:123456@localhost:5432/cafe_db?schema=public"
   ALLOW_INSECURE_COOKIES="true"
   ```
   *(مهم جداً: `ALLOW_INSECURE_COOKIES` ضروري لتتمكن من تسجيل الدخول والمحافظة على الجلسة عبر HTTP في بيئة الإنتاج).*

4. افتح موجه الأوامر (CMD) بداخل مجلد المشروع واكتب بالترتيب:
   ```cmd
   npm install
   npx prisma generate
   npx prisma db push
   set NODE_OPTIONS=--max_old_space_size=4096 && npm run build
   ```

---

## المرحلة الثالثة: إعداد خادم Nginx (بإعدادات HTTP الصارمة)

لكي يعمل النظام بكفاءة ونضمن عدم انقطاع الإشعارات المباشرة (SSE) ونمنع متصفحات الجوال من ترقية الرابط إلى `https://` تلقائياً، سنستخدم ملف إعدادات Nginx المخصص لذلك.

1. حمل برنامج Nginx للويندوز من `nginx.org` وفك الضغط عنه في `C:\nginx`.
2. انسخ ملف `nginx-windows-http.conf` الموجود في مجلد المشروع، وضعه بداخل مجلد `C:\nginx\conf` باسم `nginx.conf` (سيطلب منك استبدال الملف القديم، وافق على ذلك).
3. **للتأكد من الإعدادات**، افتح الملف ستجده يحتوي على التعليمات الخاصة ببروتوكول HTTP فقط:
   ```nginx
   server {
       listen       80;
       server_name  _; # يستقبل الاتصال من أي IP

       # إزالة أي سياسات أمان تجبر المتصفح على التحول إلى HTTPS
       add_header Strict-Transport-Security "max-age=0" always;
       proxy_hide_header Strict-Transport-Security;
       proxy_hide_header Content-Security-Policy;

       location /api/notifications/stream {
           # إعدادات خاصة للإشعارات المباشرة SSE
           proxy_pass http://127.0.0.1:3000/api/notifications/stream;
           proxy_cache off;
           proxy_buffering off;
       }

       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_set_header X-Forwarded-Proto http; # تأكيد بيئة الـ HTTP للتطبيق
           # ... باقي الإعدادات
       }
   }
   ```

## طريقة بديلة: تشغيل المشروع مباشرة على المنفذ 80 (بدون Nginx)

إذا كنت لا ترغب في استخدام Nginx وتفضل تشغيل تطبيق Next.js مباشرة ليقوم هو باستقبال الطلبات، يمكنك استخدام أمر `start:local` المُجهز مسبقاً في `package.json`.

> **تنبيه:** لا يمكنك تشغيل Nginx وتطبيق Next.js معاً على نفس المنفذ (80). إذا اخترت هذه الطريقة، تأكد من إغلاق Nginx تماماً.

1. افتح موجه الأوامر كمسؤول (CMD).
2. تأكد أن المنفذ 80 غير مستخدم من أي برنامج آخر (مثل IIS).
3. شغّل المشروع بالأمر التالي:
   ```cmd
   npm run start:local
   ```
   *هذا الأمر سيقوم بتشغيل `next start --hostname 0.0.0.0 --port 80`، مما يجعل التطبيق متاحاً لأي جهاز في الشبكة بشكل مباشر وبدون وسيط.*

إذا كنت تريد تحويل هذا الأمر إلى خدمة تعمل في الخلفية عبر أداة **NSSM** (كما سنشرح في المرحلة التالية):
- **Path:** `npm.cmd`
- **Arguments:** `run start:local`

---

## المرحلة الرابعة: التشغيل كخدمات خلفية صامتة (Windows Services)

لضمان عمل النظام تلقائياً بمجرد تشغيل السيرفر (حتى قبل إدخال كلمة مرور الويندوز):

1. **تجهيز أداة NSSM:**
   - حمل الأداة من `http://nssm.cc`.
   - فك الضغط وانتقل لمسار `win64` في موجه أوامر (CMD) كمسؤول (Administrator).

2. **خدمة خادم الويب (Next.js):**
   ```cmd
   nssm install CafeAppService
   ```
   - **Path:** اختر مسار `npm.cmd` (غالباً `C:\Program Files\nodejs\npm.cmd`).
   - **Arguments:** اكتب `run start:local` (استخدمنا start:local لتجنب أي مشاكل في المنافذ أو استماع السيرفر).
   - **Directory:** اختر مسار مشروعك `C:\cafe-services`.

3. **خدمة الوكيل (Nginx):**
   ```cmd
   nssm install NginxService
   ```
   - **Path:** اختر مسار `C:\nginx\nginx.exe`.
   - **Directory:** `C:\nginx`.

4. **تشغيل الخدمات:**
   افتح تطبيق **Services** في الويندوز، وابحث عن `CafeAppService` و `NginxService` وقم بالنقر بزر الماوس الأيمن عليهما واختر **Start**.

## إعداد جدار الحماية (Firewall) والسماح بالاتصال في الشبكة المحلية (Private Network)

بما أن الطلبات ستأتي من أجهزة الكاشير أو جوالات العملاء عبر شبكة المايكروتيك المحيطة، يجب أن نسمح للويندوز بقبول الاتصالات الخارجية على منفذ 80، خاصة إذا كانت الشبكة مصنفة كشبكة خاصة (Private Network).

1. افتح **Windows Defender Firewall with Advanced Security**.
2. اختر **Inbound Rules** من القائمة اليسرى، ثم من اليمين اختر **New Rule**.
3. اختر **Port** > **TCP** وحدد المنفذ **80** في خانة Specific local ports.
4. اختر **Allow the connection** واضغط Next.
5. **مهم جداً للشبكات المحلية:** في شاشة "Profile"، تأكد من وضع علامة (✔) بجانب **Private** و **Domain** (ويمكنك إزالة علامة Public إذا أردت تقييد الوصول للشبكة الموثوقة فقط).
6. قم بتسمية القاعدة `Cafe HTTP 80` واضغط Finish.

> **تأكيد إضافي في إعدادات الشبكة:** 
> لضمان استقرار الاتصال، تأكد من أن الويندوز يعتبر شبكة المايكروتيك المتصل بها كـ "Private Network":
> - اذهب إلى **Settings** > **Network & Internet**.
> - اضغط على خصائص شبكتك الحالية (Properties).
> - اختر **Private** بدلاً من Public. هذا يخبر الويندوز أن هذه الشبكة آمنة ويسمح باكتشاف الأجهزة والاتصال بها عبر جدار الحماية.

> **تنبيه هام للشبكة:** إذا كانت المتصفحات في الجوالات قد حفظت الرابط مسبقاً بـ HTTPS، سيحتاج المستخدمون لمسح ذاكرة التخزين المؤقت للمتصفح (Clear Cache) لمرة واحدة فقط لكي يتجاهل المتصفح أوامر التحويل السابقة ويدخل بسلاسة عبر HTTP.

الآن يمكن لأي جهاز متصل بالشبكة المحلية (المايكروتيك) فتح المتصفح وكتابة الـ IP الخاص بالسيرفر مباشرة (مثال: `http://192.168.0.153`) وسيعمل النظام بكل ميزاته وبدون شهادات SSL!
