# 📂 هيكلية المشروع ومجلداته (Project Structure)

تم تصميم هذا المشروع (Cafe Services) باستخدام **Next.js (App Router)**، وتم تقسيم المجلدات والملفات بطريقة نموذجية تضمن سهولة الصيانة وقابلية التوسع (Scalability).

فيما يلي شرح تفصيلي لهيكلية المجلدات وأهم الملفات:

## 🌲 الشجرة الرئيسية (Root Directory)

- `/.next/` : مجلد يتم إنشاؤه تلقائياً بواسطة Next.js ويحتوي على النسخ المجمعة (Build).
- `/certificates/` : يحتوي على شهادات الـ SSL المحلية (إن وجدت) لبيئة التطوير.
- `/docs/` : مجلد التوثيق الشامل (أنت تقرأ منه الآن).
- `/prisma/` : يحتوي على ملف `schema.prisma` المسؤول عن تعريف جداول قاعدة البيانات وملفات التهجير (Migrations).
- `/public/` : الملفات العامة الثابتة:
  - `/public/uploads/` : (هام جداً) هذا المجلد يستقبل جميع الملفات التي يرفعها المستخدمون (مثل صور المنتجات)، وهو مربوط بـ Docker Volume لتجنب فقدان البيانات.
  - أيقونات وخطوط (مثل `icon.svg` وغيرها).
- `/src/` : الكود المصدري الأساسي للمشروع (انظر التفاصيل أدناه).
- `.env` : ملف متغيرات البيئة الأساسية (مثل رابط قاعدة البيانات).
- `docker-compose.yml` & `Dockerfile` : ملفات تشغيل ورفع المشروع باستخدام Docker.
- `package.json` : يحتوي على المكتبات المستخدمة وأوامر التشغيل.

---

## 📁 تفاصيل مجلد `src/` (Source Code)

هذا هو القلب النابض للمشروع. تم تقسيمه إلى مجلدات فرعية حسب الوظيفة:

### 1. `src/app/` (المسارات والصفحات - App Router)
يحتوي على جميع الصفحات وواجهات برمجة التطبيقات (APIs):
- **`api/`** : مسارات الـ Backend:
  - `api/auth/` : مسارات تسجيل الدخول والتحقق من الجلسات.
  - `api/notifications/` : يحتوي على مسار `stream/route.ts` المسؤول عن فتح قناة (SSE) للاتصال اللحظي.
  - بقية المجلدات مثل `orders/`, `items/` تقوم بعمليات CRUD لقاعدة البيانات.
- **`(pages)/`** : مسارات الـ Frontend (واجهة المستخدم):
  - `(pages)/admin/` : لوحة تحكم الإدارة (لوحة الإحصائيات، الحجوزات، الطلبات).
  - `(pages)/(user-pages)/` : الواجهات الخاصة بالعميل (تصفح المنيو، عمل الطلبات).
- **`globals.css`** : ملف التنسيقات العامة للمشروع. قمنا مؤخراً بوضع متغيرات الألوان (`--color-brand`) هنا للتحكم المركزي.

### 2. `src/components/` (مكونات الواجهة - Components)
يحتوي على جميع عناصر الواجهة القابلة لإعادة الاستخدام:
- `button/` : جميع الأزرار (مثل `primary_button.tsx`).
- `icons/` : مجموعة أيقونات المشروع بصيغة SVG.
- `partials/` : أجزاء الصفحة الكبيرة (مثل القوائم المنبثقة `modals/` وقوائم الطلبات).
- مكونات أخرى مثل `metric_card.tsx` (البطاقات) و `notification_center.tsx` (مركز الإشعارات).

### 3. `src/config/` (الإعدادات المركزية - Configuration)
- `api_routes.ts` : يحتوي على روابط الـ APIs لتسهيل تعديلها من مكان واحد.
- `page_routes.ts` : روابط صفحات الـ Frontend.
- `i18n.ts` : إعدادات تعدد اللغات (عربي / إنجليزي).

### 4. `src/context/` (الحالة العامة - Context API)
يحتوي على ملفات الـ React Context لتمرير البيانات للأبناء دون الحاجة لـ Props:
- `user_context.tsx` : يخزن بيانات المستخدم الحالي.
- `reservation_context.tsx`, `order_context.tsx` : يخزن حالة الطلبات ويتم تحديثه تلقائياً عند وصول إشعار جديد عبر (SSE).

### 5. `src/lib/` (المكتبات والدوال المساعدة - Utilities)
- `prisma.ts` : اتصال قاعدة البيانات (يمنع تسرب الذاكرة بإبقاء الاتصال مفرداً).
- `emitter.ts` : محرك بث الإشعارات (Node.js EventEmitter) الداخلي.
- `validations/` : ملفات Zod للتحقق من صحة المدخلات.

### 6. `src/translations/` (الترجمة - i18n)
- `ar/` : ملفات النصوص العربية.
- `en/` : ملفات النصوص الإنجليزية.
- كل ملف يمثل قسماً (مثل `dashboard.ts` أو `validation.ts`) لتسهيل الترجمة والتعديل.
